# 修复丢包问题

## 根因分析

`Board.update()` 的调用顺序：
1. `updateInternal()` → 从 ring buffer 读取字节，解析多个数据包
2. `getNewDataInternal()` → **只返回最新一个包**
3. `PacketLossTracker.addSamples()` → 检测连续 sample index

**问题**：`updateInternal()` 每帧可能解析 N 个包（例如 packet 126-135），但 `getNewDataInternal()` 只返回 packet 135。`PacketLossTracker` 比较上一帧的 packet 125 和当前帧的 packet 135，认为中间 9 个包丢了。

这不是真正的串口丢包，而是**数据返回机制的 bug**——多包数据被丢弃了。

## 修复方案

在 `BoardCytonSerialDirect` 中添加一个**待发送队列**（`pendingDataQueue`）：

1. `processCompletePacket()` 将每个解析完成的包加入队列
2. `getNewDataInternal()` 从队列中取出**所有**待处理的包，打包成 `double[][]` 返回
3. `PacketLossTracker` 就能看到每个包的 sample index，正确检测丢包

### 关键修改

**文件**: `OpenBCI_GUI/BoardCytonSerialDirect.pde`

```java
// 新增：待发送数据队列
private ArrayList<double[]> pendingDataQueue = new ArrayList<double[]>();

// processCompletePacket() 中：
// 将 parsedEegValues + parsedAuxValues + timestamp + sampleIndex + marker
// 打包成 double[] 加入 pendingDataQueue

// getNewDataInternal() 中：
// 从 pendingDataQueue 取出所有数据，打包成 double[][] 返回
// 如果队列为空，返回 emptyData
```

### 数据格式

每个包 = 14 个 double 值：
- [0-7]: 8 个 EEG 通道 (微伏)
- [8-10]: 3 个 Aux 通道 (g)
- [11]: Timestamp
- [12]: SampleIndex (packetCount % 256)
- [13]: Marker (0)
